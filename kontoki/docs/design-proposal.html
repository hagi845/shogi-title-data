Design Proposal / SST App

「こんとき」 — こんなとき、どうする?を
まいにち1つ、れんしゅうするアプリ

ソーシャルスキルトレーニング(SST)アプリのデザイン案。対象は放課後等デイサービス・特別支援学級に通う小学生と、その支援者(指導員・教員・保護者)。子ども向け画面と支援者向け画面の2面構成で、「1日1シナリオ+きもちクイズ」だけの小さな習慣に絞る。

デザイン原則

SSTの対象児には感覚過敏・見通し不安・失敗への強い恐れを持つ子が多い。UIの全判断をこの3原則から導く。

失敗をつくらない

選択肢に「不正解」の烙印を押さない。どれを選んでも「やってみたらどうなるか」を見せて戻れる。バツ印・減点・赤色の否定フィードバックは全面禁止。

見通しが立つ

いま何問目か、次に何が起きるかを常に画面上部に予告。1回のセッションは3分以内で終わる長さに固定し、「おわり」が必ず見える。

刺激をしぼる

彩度を抑えた若葉色ベース。1画面1タスク、アニメーションは完了時のみ。ごほうび色(黄色)はスタンプにだけ使い、価値を薄めない。

キャラクター

シナリオの登場人物は固定の2人+マスコット。毎回同じ顔ぶれが登場することで、初めての状況でも「知っている世界」として安心して取り組める。イラストは太い輪郭線+ベタ塗り+白背景のミニマルタッチ(採用スタイルB)で統一し、表情の読み取りやすさを最優先する。

こども側:メインフロー

ホーム → シナリオ → けっか → きもちクイズ。文字はすべてひらがな+分かち書き、選択肢は必ず「絵+ことば」のペアで示す。

コンテンツ編成:こどもは選ばない

現場のSSTは「週1回・1テーマを深く・数週間反復」が標準。アプリはその間の6日間を埋める毎日3分の般化練習と位置づけ、編成は「週テーマ×日替わり場面」にする。選ぶ負荷はこどもに一切かけず、大人の意思決定も週1回に圧縮する。

こどもは選ばない

開いたら今日の1本が待っているだけ。選択画面は見通し原則と矛盾し、選べる子は得意なものに偏るため、メインフローから選択を排除する。

大人は「処方」する

支援者・保護者は毎日選ぶのではなく、週1回「今週のテーマ」を設定するだけ。施設のSSTセッションと同期でき、ここが支援者向け有料機能の核になる。

選ぶ楽しさは「おかわり」へ

今日の1本を終えた後だけ、過去シナリオの図書館から自由に選んで復習できる。新作は出さないので供給も破綻しない。

自動編成標準カリキュラム+実績データ(未設定ならこちら)
処方支援者・保護者が週1回設定(自動編成に優先)
▼
今週のテーマ:「かして」同じスキルを1週間反復(現場の反復原則に一致)
▼

▼
こども:きょうの1本(3分)選択なし。開いたら始まる
▽
おかわり図書館完了後のみ解放。過去のシナリオを自由に復習

支援者側:記録ダッシュボード

指導員・保護者向けはトーンを切り替え、データ密度を上げる。SSTの現場は紙の記録が主流なので、「個別支援計画に転記できる記録が自動で貯まる」ことがアプリ最大の売り。ここが有料化ポイント(施設向けB2B)。

カラーパレット

次の一歩

  1. シナリオは「スキル10種×場面バリエーション」で書く — 「かして」「じゅんばん」「ことわる」「まける」「じょうだん」など。週テーマ制なので、脚本は場面違いの量産で回る。ここがアプリの本体。
  2. MVPはこども側の①〜③だけ — きもちクイズと支援者画面は後回しでよい。1シナリオ動けば放デイ1施設でテストできる。
  3. 配布はまずTestFlight — 提携する放デイ施設の職員経由で保護者に配る。この市場なら10家族のフィードバックで十分方向が定まる。
  4. 文字はUDフォントを検討 — 実装時は BIZ UDゴシック/UDデジタル教科書体の採用を推奨(読み書き困難のある子への配慮)。
  5. 本番のイラストは発注を想定 — このモックの絵はスタイル指定用。実装時はこのタッチ(太い輪郭線・ベタ塗り・低彩度・白背景=採用スタイルB)をイラストレーターかAI画像生成に渡す。
